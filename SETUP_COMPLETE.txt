🎯 TASK WORKFLOW (8 Phases)
═══════════════════════════════════════════════════════════════════════════════

1️⃣  Created
    → Task folder initialized with all templates
    → task.md contains requirement
    → status.md shows "Created" phase

2️⃣  Planning
    → Planner analyzes requirement
    → Creates plan.md with approach & risks
    → User reviews plan

3️⃣  Contracting
    → Contractor creates contract.md
    → Defines scope, Allowed Files, Out of Scope
    → Defines acceptance criteria
    → User approves contract

4️⃣  Generating
    → Generator implements per contract
    → Creates implementation.md
    → Tracks changes in files-changed.md
    → Records decisions

5️⃣  Evaluating
    → Evaluator runs all quality checks
    → Tests: lint, typecheck, test, build
    → Creates evaluation.md with results
    → PASS or FAIL

6️⃣  Fixing (if needed)
    → If evaluation FAIL: user /resume-task to fix
    → Re-runs evaluation
    → Continues until PASS

7️⃣  Closing
    → If evaluation PASS: /close-task
    → Creates handoff.md
    → Updates PROJECT_STATE.md
    → Updates DECISIONS.md (if applicable)
    → Updates TASKS.md (status = Done)

8️⃣  Done
    → Task complete
    → Ready for next task
